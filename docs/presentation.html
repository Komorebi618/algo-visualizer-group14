为什么 MOTIVATION

学算法时的"理解断层"

教材上的算法过程，大多是几张静态插图：一个箭头、一行伪代码。 学习者写完代码只能看到「最终结果」，中间发生了什么—— 基准是怎么选的、指针怎么交换、节点怎么松弛、盘子怎么递归—— 全靠脑补。

为了解决这种「看着代码懂、合上书又忘」的问题。

我们做这样一个工具：让算法的中间状态可见、可暂停、可回放—— 把抽象的"过程"变成看得见的动画。

📖 教材静态插图无法表达"时间维度" 🧠 中间状态全靠脑补、容易出错且无法验证

03 / 10 · ARCHITECTURE

总体架构：五层 + 数据驱动渲染

分层不耦合 + 算法只产数据、不画图——这是整个项目最重要的设计心法。

🏛️ 五层架构（自上而下，上层依赖下层）

1 表现层 index.html + pages/*.html + css/style.css
2 控制层 js/*.js 中的事件绑定 与 run / reset 函数
3 业务逻辑层 js/*.js 中的纯算法实现（快排 / Dijkstra / 汉诺塔）
4 公共服务层 js/common.js：DOM 助手 + StepManager + showToast
5 数据层 data/testcases.js + 运行时 steps 数组（内存）
⛓️ 依赖规则：上层调下层，下层不感知上层。三个算法模块互不依赖。

💡 数据驱动渲染

算法只产出"状态快照"，不直接画图——让算法和 UI 完全解耦。

🧩 依赖注入

渲染回调由算法模块"注入"给 StepManager，公共层不知道画的是柱子还是节点。

📭 零外部依赖

不用 jQuery / Vue / React / CDN，完全离线可运行。

04 / 10 · CORE

核心抽象：StepManager

把它想成一个"屏幕录像机 + 播放器"——算法负责演，它负责录，播放器负责快进慢放。

🎬 录像机 + 播放器

算法运行时，每走一步就调用一次 sm.addStep({ state, description })， 把"当前是什么样"录成一帧；算法跑完，steps[] 数组里就有了一整段录像。

然后 sm.play() 自动顺序播放，sm.next() / prev() 手动单步——跟视频播放器是同一套心智模型。 

// StepManager 类骨架（js/common.js · L97-168）
class StepManager {
  addStep(step)    { this.steps.push(step); }              // 录一帧
  setSteps(steps)  { this.steps = [...steps]; ... }        // 整体替换
  next()  { this.currentIndex++; this.onStep(...); }       // 推进
  prev()  { this.currentIndex--; this.onStep(...); }       // 后退
  reset() { this.pause(); this.currentIndex = -1; }        // 复位
  play(speed) { this.isPlaying = true; this._advance(); } // 自动播
  pause() { this.isPlaying = false; clearTimeout(...); }  // 暂停
}
frame 0 / 5

🧰 关键 API 5 个

  • addStep(step) — 录入一帧
  • play(speed) — 自动播放
  • next() / prev() — 单步前进/后退
  • reset() — 回到初始
  • currentIndex — 任意跳帧

📜 自动播放引擎

js/common.js · L159-167
// StepManager 内部的自动推帧（递归 setTimeout）
_advance() {
  if (!this.isPlaying) return;
  const nextStep = this.next();   // 推进一帧
  if (!nextStep) {
    this.isPlaying = false;
    this.onPlayEnd(); return;
  }
  this.timer = setTimeout(() => this._advance(), this.speed);
}

简洁优雅：用 setTimeout 递归把自己排到下一帧 — 11 行代码就实现了完整的"自动播放 + 暂停 + 调速"。

05 / 10 · DEMO #1 · QUICKSORT

演示 ① · 快速排序

分治思想——选基准、分两边、递归排——通过柱状图把"分区"过程画出来。

🧠 分治思想：递归快排入口

选一个基准元素，把数组分成"比它小的"和"比它大的"两半， 然后对两半分别再做同样的事——这就是分治。 

// js/quicksort.js · L135-153（递归入口）
function quicksort(low, high) {
  if (low >= high) return;          // 递归终止
  const p = partition(low, high);   // ← 分（Lomuto 分区，见右下）
  quicksort(low, p - 1);             // ← 治：递归左半
  quicksort(p + 1, high);            // ← 治：递归右半
}

每一步都录一帧，柱状图上用四种颜色实时标注：

基准 (pivot) 正在比较 正在交换 已就位

▶ 现在打开演示 · 按 D 键

推荐：选「逆序数组」展示最坏情况下的递归深度

平均 O(n log n) 最坏 O(n²) 空间 O(log n)

📜 Lomuto 分区核心

js/quicksort.js · L94-102
// 单指针从左扫到右，发现比基准小的就交换
if (arr[j] <= pivotVal) {
  i++;
  if (i !== j) {
    [arr[i], arr[j]] = [arr[j], arr[i]];  // ES6 解构交换
    addStep(high, null, null, { low, high }, '交换完成');
  }
}

数据驱动渲染：算法每动一下数据，addStep 就立刻把状态快照写进 steps[]，UI 之后逐帧回放。

06 / 10 · DEMO #2 · DIJKSTRA

演示 ② · Dijkstra 最短路径

在带权图上找一个点到其他所有点的最短路——用节点染色实时呈现"已确定 / 在考察 / 未访问"。

🧠 主循环骨架：挑最小 → 松弛邻居

每一轮，从"未确定的点"里挑一个当前距离最近的， 把它标为"已确定"，然后用它去松弛它的邻居—— 如果走我能到你那里更近，那就更新。 

// js/dijkstra.js · L90-140（主循环骨架）
while (visited.length < nodes.length) {
  const u = pickMinUnvisited();   // ← 挑当前最近的未访问点
  if (u === null) break;          // 剩余不可达
  visited.push(u);                // 标记已确定
  for (const edge of adj.get(u))  // ← 松弛 u 的每条出边
    relax(u, edge);               //   ↑ 细节见右下代码卡
}

节点状态用颜色直观区分：

未访问 当前考察 松弛边 已确定

▶ 现在打开演示 · 按 D 键

推荐：选「典型 5 节点图」展示松弛 + 路径回溯

时间 O(V²) 空间 O(V + E)

📜 边松弛逻辑

js/dijkstra.js · L123-130
// 用当前点 u 去更新邻居 to 的距离
if (newDist < distances[to]) {   // 松弛判断：绕道更近就更新
  distances[to] = newDist;
  previous[to]  = u;
  steps.push(makeState(
    distances, previous, visited, u, edge, 'relax',
    `松弛边 ${u}→${to}：距离更新为 ${newDist}`
  ));
}

松弛 (Relaxation)：Dijkstra 的全部秘密就在这一行 if 判断里——"如果借道你能更近，就更新邻居的最短距离"。

07 / 10 · DEMO #3 · HANOI

演示 ③ · 汉诺塔

最经典的递归——把"n 个盘子从 A 移到 C"拆成三步走，最直观的递归动画。

🧠 递归三步走

要把 n 个盘子 从 A 移到 C，递归地做三件事：

  • ① 把上面 n−1 个盘子借助 C 移到 B
  • ② 把最大的盘子从 A 移到 C
  • ③ 把那 n−1 个盘子从 B 借助 A 移到 C
初始：A [▰▰▰] B [ ] C [ ] ↓ 递归三步走，共 2ⁿ − 1 次移动 完成：A [ ] B [ ] C [▰▰▰]

一次调用拆三步，每步再递归——总步数恰好是 2ⁿ − 1：

n=1 → 1 步 n=3 → 7 步 n=6 → 63 步 n=8 → 255 步

▶ 现在打开演示 · 按 D 键

推荐：从 3 盘开始（7 步），再切 6 盘（63 步）感受指数增长

时间 O(2ⁿ) 空间 O(n) 2ⁿ − 1 步

📜 递归三步曲

js/hanoi.js · L122-131
// 把 k 个盘子从 from 借助 via 移到 to
const hanoi = (k, from, to, via) => {
  if (k <= 0) return;
  if (k === 1) { moveOne(from, to); return; }
  hanoi(k - 1, from, via, to);   // ① 先把上面 n-1 搬开
  moveOne(from, to);              // ② 搬最大那一个
  hanoi(k - 1, via, to, from);   // ③ 再把 n-1 搬回来
};

递归之美：9 行代码描述了一个 2ⁿ − 1 步的过程——分解、解决、合并，递归思维的教科书示范。
