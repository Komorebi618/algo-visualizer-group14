# 项目要求

## 基本要求

1. 算法选择：3个算法

   - 快速排序：柱状图交换动画

   - Dijkstra最短路径：节点图逐步染色

   - 汉诺塔：柱子加盘子

提示：项目靠考虑算法难度和Canvas显示难度

2. 使用统一入口和统一界面风格

3. 数据：每个算法支持手动输入数据或随机生成测试数据，至少2个测试用例，项目总用例不少于6条

4. 显示：

   1. 当前的输入数据

   2. 执行过程以及其中的关键状态变化、每一步的说明

   3. 最终执行结果

   4. 时间复杂度和空间复杂度说明

5. 文稿：提交 `README.md`，说明系统运行方式、算法选择方式、输入格式和测试方式。

## 拓展需求（选做）

1. 增加第4种类型算法

2. 展示控制功能：单步执行、自动播放、暂停、重置等基本控制；上一步回退

3. 导出算法日志

4. 同类算法对比

5. 用户保存自定义测试数据

6. 错误输入提示与输入范围检查

7. 题目讲解，关键步骤解释文字

# 成果提交

## 4\.1 小组阶段文档要求（word指导书原文）

项目组应提交需求分析文档、系统设计文档和最终项目文档。三类文档应相互对应，不能前后矛盾。

### 4\.1\.1 需求分析文档

需求分析文档应按照 “问题定义、可行性研究、需求分析”的基本思路，说明系统要解决什么问题、服务什么用户、完成哪些功能、满足哪些非功能特性等。应使用用例图、活动图、数据流图、E\-R图等加以描述。要求体现AI 辅助需求分析的使用说明。

### 4\.1\.2 系统设计文档

系统设计文档应按照 “总体设计”和“详细设计”的基本思路，说明系统如何由需求转化为可实现的软件结构。应包含系统结构设计、各模块处理过程设计及数据库设计、接口设计等内容，并选择使用系统架构图、包图、类图、时序图、流程图等加以描述。要求体现AI 辅助系统设计的使用说明。

### 4\.1\.3 最终项目文档

最终项目文档是小组最终提交的完整技术文档，应包括需求分析、系统设计、系统实现和系统测试四个部分。最终项目文档应以最终完成的系统为依据，对前期需求和设计进行必要修订，说明实际实现情况和测试结果。最终项目文档应体现完整软件开发过程：从问题定义和需求分析出发，经过总体设计、详细设计和编码实现，最后通过软件测试验证系统是否满足需求。

要求在最终项目文档中体现AI辅助开发说明：小组使用了哪些AI工具，AI参与了哪些环节，哪些内容经过人工修改或验证。

## 4\.2 个人开发日志要求

每位学生必须提交个人开发日志。个人开发日志应真实记录本人在课程设计中的工作过程，不得由小组统一代写。

个人开发日志建议包含以下内容：

1）姓名、学号、小组编号、项目题目

2）本人负责的模块或任务

3）每日或阶段性完成内容

4）遇到的问题

5）问题分析和解决过程

6）参与的测试或文档工作

7）与其他成员协作的情况

8）AI 使用记录

9）个人总结

AI 使用记录必须体现以下信息：

| **日期** | **AI 工具与模型**       | **使用目的** | **Prompt或问题摘要**   | **AI输出用途** | **本人修改或验证** | **对应文件或功能** |
| -------- | ----------------------- | ------------ | ---------------------- | -------------- | ------------------ | ------------------ |
| 第 5 天  | ChatGPT/Codex/Claude 等 | 调试报错     | 分析某模块运行错误原因 | 部分采用       | 修改代码并重新测试 | src/\.\.\.         |

每位学生至少记录5条有效AI使用过程，其中至少包括：

1）1条需求分析或系统设计相关记录。

2）2条代码实现、调试或重构相关记录。

3）1条测试用例生成或测试分析相关记录。

4）1条 AI 输出被修改、否定或重新验证的记录。

注意：不要求提交完整冗长的AI聊天原文，但应保留关键对话片段、截图或摘要，能够说明AI给出了什么建议以及学生如何处理。
