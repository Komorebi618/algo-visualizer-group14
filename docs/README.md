## 答辩演示稿（presentation.html）使用说明

**打开方式**：双击 `presentation.html`，浏览器以单页方式渲染。

**翻页**：
- ← / → / Space — 上一页 / 下一页
- 数字 `1`~`9` / `0` — 直跳第 1~9 / 第 10 页
- `Home` / `End` — 首末页
- `G` — 打开跳转面板，按页码或关键字（如「Dijkstra」「快排」「AI」）搜索
- 顶部分段进度条任一段点击即跳

**演示交互**：在 Slide 5/6/7（快排 / Dijkstra / 汉诺塔）按 `D` 键或点击橙色提示卡，在新标签打开对应算法的真实运行页面（`../pages/*.html`），可现场切换数据 / 拖动进度条演示。

**幻灯片大纲**：

| # | 标题 | 主要内容 |
|---|------|----------|
| 1 | 封面 | 项目名 · 团队 · 日期 |
| 2 | 选题动机 × 项目范围 | 为什么做 · 做什么 · 不做什么 |
| 3 | 总体架构 | 五层分层 + 数据驱动渲染 |
| 4 | 核心抽象 StepManager | 录像机 + 播放器比喻 + 类骨架 + 自动播放引擎源码 |
| 5 | 演示 ① 快速排序 | 分治思想 + 递归入口源码 + Lomuto 分区源码 |
| 6 | 演示 ② Dijkstra | 主循环骨架 + 边松弛源码 |
| 7 | 演示 ③ 汉诺塔 | 三步走 + ASCII 塔图 + 递归源码 |
| 8 | 工程实践 | Git 多分支协作 SVG 图 + 验收清单 + 测试用例 + 浏览器兼容 |
| 9 | AI 协作辅助开发 | 课程指导书鼓励 + 我们的 review / 删冗余 / 对齐文档 三件事 |
| 10 | 总结与展望 | 已交付 + 后续可做 + 三人致谢 |

**视觉风格**：浅色 Mesh Gradient 背景 + 毛玻璃卡片 + Bento Grid 布局，与项目首页 [index.html](../index.html) 保持一致。

**应急 PDF 备份**：在浏览器中 `Ctrl+P` → 保存为 PDF，可作为 U 盘失灵时的应急稿。

## 文档维护规则

- `.docx` 文件改动须本地 Word 保存后整体替换，不在 Git 中做行级 diff
- `presentation.html` 的所有 CSS 设计 token 复用项目 [css/style.css](../css/style.css)，不要硬编码色值；新增样式仅追加内联 `<style>`
- 三份 `.docx` 与项目根的 [CLAUDE.md](../CLAUDE.md) 在校对日志上保持同步——文档更新后在 CLAUDE.md 末尾追加一行记录
