# algo-visualizer-group14

算法过程可视化系统,基于 HTML5 + CSS3 + JavaScript(ES6) + Canvas API 实现,包含**快速排序**、**Dijkstra 最短路径**和**汉诺塔**三个算法的可视化模块。

纯前端方案,**无构建工具、无框架、无后端、无 CDN 依赖**,完全离线可运行。

---

## 语言与编译器版本

| 项目 | 要求 |
|------|------|
| HTML | HTML5 标准 |
| CSS | CSS3(含 CSS 变量、CSS Grid、Flexbox、`backdrop-filter`) |
| JavaScript | ES6+(箭头函数、解构、模板字符串、`class`、`const`/`let`、可选链) |
| 运行环境 | 现代浏览器,无需任何编译器或构建工具 |
| 推荐浏览器 | **Chrome ≥ 99** / **Edge ≥ 99** / **Firefox ≥ 103**(均已实测通过) |

> 项目使用了 `ctx.roundRect`(Chrome 99+) 和 `backdrop-filter`(Firefox 103+),已为旧版浏览器做了降级处理(`ctx.rect` 兜底)。

## 系统运行方式

本项目**不需要任何安装或编译**,有以下两种方式启动:

### 方式 1:直接双击(最简单)

1. 解压或克隆本仓库到本地
2. 双击 [`index.html`](index.html) 在默认浏览器打开
3. 即可进入首页,点击任一算法卡片进入对应模块

```bash
git clone https://github.com/Komorebi618/algo-visualizer-group14.git
cd algo-visualizer-group14
# 双击 index.html(或在浏览器地址栏粘贴绝对路径)
```

### 方式 2:启动本地服务器(推荐)

某些浏览器对 `file://` 协议有 CORS 限制(如不允许加载 `data/testcases.js`),此时建议用任意静态服务器:

```bash
# 方式 a:Python 3 自带 http.server(推荐,无需额外依赖)
cd algo-visualizer-group14
python -m http.server 8000
# 浏览器访问 http://localhost:8000

# 方式 b:VS Code Live Server 插件,右键 index.html → Open with Live Server
```

## 输入格式

每个算法页面左侧面板的"**输入数据**"区域用于配置算法输入,**三种输入方式**均支持:

| 输入方式 | 触发位置 | 说明 |
|----------|----------|------|
| 手动输入 | 文本框/数字框 | 按格式输入后点"开始/运行"按钮 |
| 随机生成 | 「随机生成」按钮 | 在合法范围内随机生成数据,填入输入框 |
| 预设用例 | 「预设测试用例」下拉框 | 切换下拉项,数据自动填入输入框 |

### 快速排序输入格式

- **输入框**:`手动输入数组`
- **格式**:数字以**逗号、空格、中文逗号或分号**分隔
  - 例:`7, 2, 9, 4, 3, 8` 或 `7 2 9 4 3 8` 或 `7;2;9;4;3;8`
- **约束**:
  - 元素数量:**2 ~ 20** 个
  - 每个元素:**1 ~ 99** 的整数
- **示例**:`5, 3, 8, 1, 4`

### Dijkstra 输入格式

- **节点数**:2 ~ 10 的整数(数字框 `节点数`)
- **源节点**:从下拉框选择(根据节点数动态生成 `节点 0` ~ `节点 N-1`)
- **边列表**:textarea 中**每行一条边**,格式 `起点 终点 权重`,空格分隔
  - 例:
    ```
    0 1 4
    0 2 1
    2 1 2
    1 3 1
    ```
- **约束**:
  - 起点/终点为 `0 ~ 节点数-1` 的整数
  - 权重为**正整数**(不支持 0 或负数)
  - 支持**有向图**(`A→B` 不自动等价于 `B→A`,如需双向需各写一行)

### 汉诺塔输入格式

- **输入框**:`盘子数(1 ~ 8)`,只接受**整数**
- **约束**:`1 ≤ 盘子数 ≤ 8`(超过 8 步数指数爆炸,影响动画流畅度)
- **示例**:`3`(对应经典 7 步移动)

### 错误提示

非法输入会触发**左上角 Toast 浮层**(自动 3.5s 消失),例如:
- "请输入数组数据"
- "节点数须为 2~10 的整数"
- "边权重须为正整数"
- "盘子数必须在 1 到 8 之间。"

输入校验全部在 JS 层完成,系统**不会因非法输入崩溃**。

## 测试方式

### 预置测试用例(共 9 条,满足课设要求 ≥ 6 条)

所有用例集中在 [`data/testcases.js`](data/testcases.js) 维护,通过 `window.Testcases` 暴露给各算法模块。

| 算法 | 用例数 | 用例名 | 用途 |
|------|--------|--------|------|
| 快速排序 | **3** | 随机整数数组 / 包含重复元素 / 逆序数组 | 基础分区 / Lomuto 重复处理 / 接近 O(n²) 最坏情况 |
| Dijkstra | **2** | 小型有向图 / 稀疏图 | 典型路径松弛 / 不同拓扑下的路径发现 |
| 汉诺塔 | **4** | 1 盘 / 3 盘 / 4 盘 / 6 盘 | 边界 / 经典 / 中等 / 指数增长 |

切换下拉框选项,数据自动填入输入框,再点"开始/运行"即可演示。

### 手动测试步骤

每个算法页面的标准测试流程:

1. **选择测试用例**(下拉框)或**手动输入**或点**「随机生成」**
2. 点击「**▶ 开始排序 / ▶ 运行**」(主操作按钮)
3. 观察 Canvas 自动播放,每步同步:
   - **当前步骤**卡片显示文字说明
   - **进度条** + 计数器(N / 总步数)实时更新
   - **步骤日志**累加记录每一帧
4. 通过**播放控制条**手动控制:
   - `⏮` 回到第一步
   - `◀` 上一步
   - `▶/⏸` 播放/暂停
   - `⏭` 跳到末尾
   - 进度条点击任意位置跳转到对应帧
   - 快速排序额外提供**速度滑块**(1=最慢 1200ms,10=最快 120ms)
5. 点「**↺ 重置**」清空当前播放状态

### 异常输入测试

每个算法都做了输入边界检查,可故意输入非法值验证 Toast 提示:

| 测试场景 | 预期 Toast |
|----------|------------|
| 快速排序输入空 | "请输入数组数据" |
| 快速排序输入 `1, 2, abc` | "无效数字:abc" |
| 快速排序输入 `1, 100` | "每个数字需为 1 - 99 的整数" |
| 快速排序输入 `5` | "至少需要 2 个数字" |
| Dijkstra 边列表为空 | "请输入节点数和边列表" |
| Dijkstra 输入 `0 1 -3` | "边权重须为正整数" |
| Dijkstra 节点编号超范围 | "节点编号 X 超出范围(0 ~ N-1)" |
| 汉诺塔输入 `0` 或 `9` | "盘子数必须在 1 到 8 之间。" |
| 汉诺塔输入 `abc` | "请输入有效的整数。" |

## 项目基础信息

### 已实现功能

1. **三种算法可视化**
   - 快速排序:Lomuto 分区柱状图(基准/比较/交换/已就位四色标记)
   - Dijkstra 最短路径:节点逐步染色 + 最短路径表
   - 汉诺塔:三柱递归动画 + 移动次数统计
2. **统一入口与界面风格**:首页 + 三页统一的浅色 + 便当盒(Bento Grid)布局 + 毛玻璃质感
3. **三种数据输入方式**:手动输入 / 随机生成 / 预设用例(共 9 条)
4. **完整可视化展示**:
   - 当前输入数据
   - 执行过程中关键状态变化与每步文字说明
   - 最终执行结果
   - 时间复杂度和空间复杂度
5. **完整播放控制**:⏮ ◀ ▶/⏸ ⏭ + 进度条点击跳转 + 重置 + 速度滑块(快排)
6. **错误处理**:左上角 Toast 浮层,非法输入不会导致系统崩溃

### 不实现的拓展功能(本期不做)

- 第 4 种类型算法
- 算法日志导出
- 同类算法对比
- 用户保存自定义测试数据(LocalStorage)
- 题目讲解 / 关键步骤解释文字
